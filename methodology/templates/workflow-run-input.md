# Request — Existing Run (Funnel B)

Use this to ask for a normal run of a workflow we already produce for you. It feeds
`/run-workflow`. Keep it short — this is just the inputs for one run.

## 1. Which workflow?
The workflow name, or just describe the deliverable you want produced.

## 2. Run date / period
Which week, month, or date does this run cover?

## 3. Inputs
Attach the input files, or paste the source material for this run.

## 4. One-time notes (this run only)
Anything special for *just this run* — an extra section, a different emphasis, a
file to fold in once. This does **not** change future runs. If you want a permanent
change, that's a revision instead (see `workflow-revision-request.md`).

## 5. Deadline
When do you need the finished output?

## 6. Sensitivity confirmation
Confirm the sensitivity of the inputs (e.g. internal / confidential / restricted).
Raw confidential or restricted content is handled locally and never committed.
